Counter and accumulator screening

Storage item Projected maximum Type limit Status

Histogram screening assumes every update could hit one bin. Copula screening uses N × max(|z|max, |z|max²), a conservative magnitude bound rather than a proof of numerical accuracy.

Methodology

What the model means

01

Why 1D histograms are not enough

A histogram records how often each value range occurs for one signal. It cannot reconstruct whether high or low values in different signals occurred at the same time.

T1 high + T2 high at the same instant → not recoverable
02

What copula accumulators retain

Each current signal vector is mapped to normalized values. Pairwise products are accumulated without storing historical raw samples.

x[i] → bin k[i] → z[i] → sum_zz[i,j] += z[i] · z[j]
03

The deliberate information trade-off

A full multidimensional grid needs Bd cells. The copula model reduces this to d(d + 1)/2 pair entries. It is compact, but it does not retain every higher-order dependency.

—
Boundaries of this estimate

Assumptions and limits

  • No historical raw data is stored or simulated.
  • Copula accumulators provide a compact pairwise dependency summary, not a full joint distribution.
  • The t-copula uses a fixed, calibrated ν. Live estimation of ν is outside this model.
  • CPU cycle values are editable engineering assumptions and require calibration on target hardware.
  • A disabled LUT uses a user-supplied transform cost; this page does not implement an inverse CDF.
  • Accumulator screening is conservative and does not replace fixed-point range and precision analysis.
  • NVM write cycles, wear leveling, AUTOSAR integration, and scheduling overhead are not modeled.
